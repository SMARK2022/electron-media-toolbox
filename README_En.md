## 📖 Introduction

**SMARK Media Tools** is a local media management tool designed for photographers. It provides an efficient automated solution for organizing large batches of photos generated by continuous shooting and handling duplicate images:

1.  **Smart Grouping**: Automatically clusters burst sequences and similar scenes using HSV histogram similarity. Threshold is adjustable via slider with precise input support.
2.  **Aesthetic Scoring**: Integrates ZJU's **LAR-IQA** (No-Reference Image Quality Assessment) algorithm, accelerated by ONNX Runtime / DirectML, to automatically select the best photos in each group.
3.  **Face Detection & Eye Analysis**: Integrates three ONNX models — det_10g (face detection) + ocec_l (eye open/closed classification) + 2d106det (facial landmarks). Faces are color-coded as open (green) / suspicious (amber) / closed (red) to help identify rejects.
4.  **Retention Policy**: Auto-disables redundant shots by IQA ranking or eye-open priority, with configurable keep count per group.
5.  **Quick Export**: Rapidly mark photos for retention or deletion, and export selected photos (with matching RAW files) to your target folder.

> **Performance**: ~3ms/frame for thumbnails, ~1s/frame for AI inference (GPU dependent). Preview uses ImageBitmap caching — previously viewed photos switch with zero latency.

## ✨ Key Features

- **⚡ Full-Stack Acceleration**
  - **ONNX Runtime (DirectML)**: Exports the LAR-IQA model to ONNX format with DirectML backend, providing unified support for NVIDIA/AMD/Integrated GPUs and CPU on Windows. Accelerates inference while eliminating heavy PyTorch dependencies and CUDA requirements. (macOS runs on CPU for now.)
  - **Nuitka Compilation**: The Python backend is compiled into a single standalone binary (`web_api.exe` on Windows / `web_api.app` on macOS), ensuring fast startup and low resource overhead.

- **🧠 Automated Backend Management**
  - The Electron main process automatically manages the backend lifecycle (start, keep-alive, shutdown).
  - Built-in health checks: Probes backend status on launch and monitors latency in real-time.

- **📦 Ready to Use**
  - Windows: `.msi` / `.exe` installer. macOS: `.zip` package. Both contain the Electron app, pre-compiled backend, and ONNX models.
  - **No Python environment configuration required**. No manual CUDA toolkit installation needed.

- **🎨 Modern UX**
  - Built with Shadcn UI + Tailwind CSS with keyboard shortcut support.
  - Intuitive "Group & Detail" view with double-click to toggle enabled/disabled, arrow keys to navigate, scroll to zoom, drag to pan.
  - Face thumbnail strip with click-to-focus and auto-tracking mode.

## 📥 Installation

### 1. For End Users (Recommended)

Download the latest installer from the [Releases](../../releases) page:

- **Windows**: Download `.msi` or `.exe`, simply double-click to install. No additional configuration required.
- **macOS** (Experimental): Download `.zip`, extract and drag to Applications. Uses ImageIO / QuickLook for thumbnail acceleration.

> Windows SmartScreen may block the unsigned installer. Click "Run anyway" to proceed.

### 2. For Developers (Source)

Requirements: Node.js >= 22 and Python 3.12+ (Conda recommended).

```bash
# 1. Clone the repository (requires Git LFS for ONNX models ~105MB)
git clone https://github.com/SMARK2022/electron-media-toolbox.git
cd electron-media-toolbox
git lfs install
git lfs pull

# 2. Setup backend (Conda/venv recommended)
cd python
pip install -r requirements.txt
cd ..

# 3. Install frontend dependencies + compile native module
npm install
npm run rebuild          # Compile better-sqlite3 for Electron ABI

# 4. Compile backend (requires Conda env with Nuitka)
npm run python:make      # Output in python/out/

# 5. Start the app
npm run start
```

> **Note**: In dev mode, Electron auto-starts the compiled backend. If the backend binary is missing, the frontend still launches but with limited functionality.

### Testing

```bash
npm run test             # Unit tests (vitest, jsdom, cross-platform)
npm run test:e2e         # E2E tests (Playwright + Electron, requires npm run package first)
npm run lint             # ESLint
npx tsc --noEmit         # TypeScript type check
npm run format           # Prettier format check
```

## 🛠️ Tech Stack

| Component        | Tech Choice                                      | Description                                                                 |
| :--------------- | :----------------------------------------------- | :-------------------------------------------------------------------------- |
| **UI / Desktop** | Electron 39, Vite, React 19, TypeScript (strict) | Shadcn UI + Tailwind CSS v4 + Zustand + TanStack Router/Query               |
| **Backend**      | FastAPI, Uvicorn                                 | Core business logic and file I/O, compiled to standalone binary             |
| **AI Inference** | **ONNX Runtime (DirectML)**                      | LAR-IQA + face detection/eye/landmark models, GPU on Windows / CPU on macOS |
| **Database**     | better-sqlite3 (WAL)                             | Cross-process concurrent read/write between Electron and Python             |
| **Packaging**    | Electron Forge 7                                 | Windows: Squirrel `.exe` + WiX `.msi` / macOS: `.zip`                       |
| **Testing**      | Vitest + Playwright                              | Unit tests (jsdom) + E2E tests (Electron integration, 53 cases)             |
| **CI/CD**        | GitHub Actions                                   | quality → build → e2e → release unified pipeline                            |

## 📝 Changelog

### v2.1.3 (2026-06-25)

- **macOS multi-platform support**: Native thumbnail acceleration (IShellItemImageFactory / ImageIO + QuickLook), Nuitka `--mode=app` compilation into `.app` bundle.
- **Configurable retention policy**: Supports IQA ranking / eye-open priority criteria, adjustable keep count per group.
- **Precise similarity threshold input**: Slider with direct numeric input.
- **SQLite concurrency safety**: WAL mode + atomic clearPhotos transaction + import guard during detection.
- **EXIF reading optimization**: 64KB partial buffer instead of full file read, significantly reducing I/O overhead.
- **Preview loading optimization**: ImageBitmap cache + canvas rendering, zero-latency switching for cached photos.
- **E2E test suite**: Playwright + Electron, 53 test cases covering import → filter → toggle → export.
- **Unified CI/CD pipeline**: quality → build → e2e → release dependency chain.
- **i18n & accessibility fixes**: Missing translations, aria-labels, dead code cleanup.

- **E2E test framework**: Playwright + Electron, covering import → filter → toggle → export workflow.
- **CI/CD infrastructure**: GitHub Actions automation (test / build / release), Git LFS for ONNX models.
